# Oracle - The Central Inventory (OraInventory)

Official Link
	|_ 1. [Link 1](https://docs.oracle.com/en/database/oracle/oracle-database/26/cwlin/about-the-oracle-inventory-directory-and-installation.html?utm_source=chatgpt.com)

The directory which is called "oraInventory" stores an inventory of all Oracle software installed on the system.

Focus here.
The oraInventory is a "directory". Oracle refers to this directory itself as the Central Inventory.

This directory contains files that make up that inventory, including an important file:
`ContentsXML/inventory.xml`

This file `inventory.xml' contains inventory data (such as registered Oracle Homes).

``` bash
oracle@10:~> cd $ORACLE_BASE/../oraInventory
oracle@10:/u01/app/oraInventory> ls -lrt
```

There are other files & directories as well that reside in the central inventory directory, for example:

``` bash
[dir] ContentsXML/ — stores core inventory metadata
		[file] inventory.xml — registered Oracle Homes
		[file] comps.xml — installed components
		[file] libs.xml — library metadata
		[file] oui-patch.xml — patch-related metadata
[dir] logs/ — installer and patch logs
[dir] backup/ — inventory backup information
[dir] oui/ — Oracle Universal Installer-related files
```

Some additional files may appear inside the oraInventory directory, only during certain operations and are transient in nature. Because they are created only when needed, they may not exist permanently inside the oraInventory directory.

For example, lock-related files may appear during software installation or patching to prevent multiple processes from updating the inventory simultaneously.
