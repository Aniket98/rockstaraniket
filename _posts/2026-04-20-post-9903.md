### Talking about - The Central Inventory (OraInventory)

The directory which is called "oraInventory" stores an inventory of all Oracle software installed on the system.

Focus here.
The oraInventory is a "directory". Oracle refers to this directory itself as the Central Inventory.

This directory `oraInventory` contains files that make up that inventory, including an important file:
`ContentsXML/inventory.xml`

This file `inventory.xml' contains inventory data (such as registered Oracle Homes).

There are other files & directories as well that reside in the central inventory directory, as shown below:

``` bash
[dir] oraInventory
		|_ [dir] ContentsXML/ — stores core inventory metadata
		|		|_ [file] inventory.xml — registered Oracle Homes
		|		|_ [file] comps.xml — installed components
		|		|_ [file] libs.xml — library metadata
		|		|_ [file] oui-patch.xml — patch-related metadata
		|_ [dir] logs/ — installer and patch logs
		|_ [dir] backup/ — inventory backup information
		|_ [dir] oui/ — Oracle Universal Installer-related files
		|_ [file] orainstRoot.sh
		|_ [file] oraInst.loc
```

Some additional files may appear inside the oraInventory directory, only during certain operations and are transient in nature. Because they are created only when needed, they may not exist permanently inside the oraInventory directory.

For example, lock-related files may appear during software installation or patching to prevent multiple processes from updating the inventory simultaneously.

----------- Paras ----------------

Inventory information is stored in Extensible Markup Language (XML) format. The XML format enables easier diagnosis of problems and faster loading of data. Any secure information is not stored directly in the central inventory. As a result, during removal of some products, you may be prompted to enter the required credentials for validation.

Oracle recommends placing the central inventory on a local file system that is not shared by other systems, since the central inventory is a system-specific inventory of the installations on this system. It is strongly recommended that you place the central inventory on a local disk so that installations from other systems do not corrupt the inventory. You should not place the central inventory in any Oracle home, Domain home or Application home location.
