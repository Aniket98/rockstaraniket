# Linux - Useful Commands

A collection of practial linux commands.
I will keep updating this same post time-to-time with new commands or information.

### SUSE Commands

------------------------------ zypper package search ----------------------------

-- search package in enabled repos 

``` bash
zypper se -s packagename  
zypper se -s packagename=16.5  
```

-- search package in all repos 

``` bash
zypper search-packages packagename  
zypper search-packages -s packagename  
```

------------------------------ zypper repositories ----------------------------

-- see which repos are enabled or disabled

``` bash
zypper lr -u | awk -F'|' '{gsub(/ /,"",$4); print $2 " -> " ($4=="Yes"?"Enabled":"Disabled")}'  
```

### General LINUX Commands

#### --------------------- move and copy: mv & cp 

-- renaming or moving files (always use -i option when doing manual renaming)

``` bash
mv -i current_filename target_path/new_filename
```

note: Use cp also with -i option.

note: The -i option does not display warning for folders, because if folder name already exists, it will move your whole folder inside target folder. So, during folder RENAMING or MOVING, pay attention because -i won't save you here! Although, you can move back folder back to original location, but depending on situation it can doom you if not careful💀.

For folder renaming, better check if same new name exists using: ls -lrt|grep ^d
To only list REGULAR files: ls -lrt|grep ^-
To only list SYMLINKs: ls -lrt|grep ^l
Here, ^ means start of line (regex).

Also, use mv -t option to set target directory explicitly, it makes syntax neat, and hence avoids visual ambiguity.

example:

``` bash
mv -i file1 -t targetpath/
mv -i file1 file2 -t targetpath/
```

note: Always use / at end of TARGET folder name.

Also, while moving or renaming multiple files, you can use verbose -v option to print operation details.

``` bash
mv -iv file1 file2 file3 -t targetpath/
```

#### --------------------- LOFS - Loopback File System (a virtual FS)

Create a directory in / and use it as FS.
Below example is shown on Rocky Linux 8.10:

1. Create a directory to hold the file-backed FS
mkdir -p /mnt/mssql_tempdb

2. Create a 40 MB file
dd if=/dev/zero of=/mnt/mssql_tempdb/tempdb.img bs=1M count=40

3. Format it as XFS (or ext4 if you prefer)
mkfs.xfs /mnt/mssql_tempdb/tempdb.img

4. Mount the file as a loop device
mount -o loop /mnt/mssql_tempdb/tempdb.img /mnt/mssql_tempdb

5. Verify it’s mounted
df -h /mnt/mssql_tempdb

6. Make it permanent in /etc/fstab
echo "/mnt/mssql_tempdb/tempdb.img /mnt/mssql_tempdb xfs loop 0 0" >> /etc/fstab
