# SQL Server - Installing on Linux

Installing SQL Server on Rocky Linux.

OS: Rocky Linux 8.10
Database: SQL Server 2022 (16.x)

Since, I come from a linux strong background, I am first installing SQL Server on Linux.
It gives me edge as I am already aware of the Linux Environment.

I can learn sql server database much easier this way and become comfortable to it.
Later, we will also install on Windows OS.

Official Link: https://learn.microsoft.com/en-us/sql/linux/quickstart-install-connect-red-hat?view=sql-server-linux-ver16&preserve-view=true&tabs=rhel8%2C2025rhel9

--- configure
sudo /opt/mssql/bin/mssql-conf setup

After the package installation finishes, run mssql-conf setup by using its full path. Follow the prompts to set the sa password and choose your edition. As a reminder, the following SQL Server editions are freely licensed: Evaluation, Developer, and Express.

Note: You must have atleast 2GB Memory, else the setup will fail.

Caution : Your password should follow the SQL Server default password policy. By default, the password must be at least eight characters long and contain characters from three of the following four sets: uppercase letters, lowercase letters, base-10 digits, and symbols. Passwords can be up to 128 characters long. Use passwords that are as long and complex as possible.

--- check mssql service (start if not started) (OPTIONAL: Enable service)
systemctl status mssql-server

--- allow remote connections by allowing port through firewall if present
sudo firewall-cmd --zone=public --add-port=1433/tcp --permanent
sudo firewall-cmd --reload

----------------------------- Install sqlcmd utility ----------------------

--- download repo
curl https://packages.microsoft.com/config/rhel/8/prod.repo | sudo tee /etc/yum.repos.d/mssql-release.repo

--- remove any old version of mssql-tools (if any)
sudo yum remove mssql-tools unixODBC-utf16 unixODBC-utf16-devel

--- install mssql-tools18
sudo yum install -y mssql-tools18 unixODBC-devel

--- to update to latest version of mssql-tool18
sudo yum check-update
sudo yum update mssql-tools18

--- for LOGIN shells - export bash profile
echo 'export PATH="$PATH:/opt/mssql-tools18/bin"' >> ~/.bash_profile
source ~/.bash_profile

--- for INTERACTIVE/NON-LOGIN shells - export bashrc
echo 'export PATH="$PATH:/opt/mssql-tools18/bin"' >> ~/.bashrc
source ~/.bashrc

--- connect to sqlserver instance
sqlcmd -S localhost -U sa

--- connect to sqlserver instance (to make SSL encryption optional)
sqlcmd -S localhost -U sa -No

Note: If you only try "sqlcmd -No" it will attempt OS level user authentication, instead of picking default user as "sa"

--- connect to sqlserver instance (by explicity setting the protocol:servername,port)
sqlcmd -S tcp:localhost,1433 -U sa -No

--- to pass in line T-SQL
sqlcmd -S localhost -U sa -Q 'select @@VERSION'
