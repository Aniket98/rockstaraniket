# SQL Server - Installing on Linux

Installing SQL Server on Rocky Linux.

OS: Rocky Linux 8.10
Database: SQL Server 2022 (16.x)

Since, I come from a linux strong background, I am first installing SQL Server on Linux.
It gives me edge as I am already aware of the Linux Environment.

I can learn sql server database much easier this way and become comfortable to it.
Later, we will also install on Windows OS.

Official Link: https://learn.microsoft.com/en-us/sql/linux/quickstart-install-connect-red-hat?view=sql-server-linux-ver16&preserve-view=true&tabs=rhel8%2C2025rhel9

Note: The below procedure focuses mainly on installation for now. Configuring the sql server from various perspectives (like custom data directory location, memory configurations, security configurations, etc are not discussed in this article. We can do that in future posts. Thanks.)

sudo curl -o /etc/yum.repos.d/mssql-server.repo https://packages.microsoft.com/config/rhel/8/mssql-server-2022.repo

--- connect to sqlserver instance (to make SSL encryption optional)
sqlcmd -S localhost -U sa -No

Here, we connect to the SQL Server without SSL mode by making it optional using flag '-No'.
After successful DB Connection, we list the SQL Server databases list as well.

Remember: SQL Server processes T-SQL in batches. In client tools like SSMS or sqlcmd, the GO statement is used as a client-side batch separator to send each batch for execution.

``` bash
[mssql@rocky810 ~]$ sqlcmd -S localhost -U sa -No
Password:
1> select name from sys.databases;
2> go
name
--------------------------------------------------------------------------------------------------------------------------------
master
tempdb
model
msdb

(4 rows affected)
1>
```
